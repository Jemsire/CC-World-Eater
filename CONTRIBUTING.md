# Contributing to CC-World-Eater

Thank you for your interest in contributing to CC-World-Eater! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/Jemsire/CC-World-Eater.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`

## Development Setup

1. Set up a Minecraft server/client with CC Tweaked
2. Test your changes in-game before submitting a PR
3. Ensure compatibility with both Minecraft 1.12 and 1.16+ if possible

## Code Style

- Follow existing Lua code style conventions
- Use meaningful variable and function names
- Comment complex logic
- Keep functions focused and modular

## Testing

Before submitting a PR, please test:

- [ ] Installation works correctly
- [ ] Hub computer starts and runs properly
- [ ] Turtles connect and respond to commands
- [ ] Mining operations work as expected
- [ ] Works with and without peripheral mods (if applicable)
- [ ] Single and dual-drive setups work

## Submitting Changes

1. Commit your changes with clear, descriptive messages
2. Push to your fork: `git push origin feature/your-feature-name`
3. Open a Pull Request with:
   - Clear description of changes
   - Testing information
   - Any breaking changes noted

## Reporting Bugs

Use the bug report template and include:
- Minecraft and mod versions
- Steps to reproduce
- Expected vs actual behavior
- GPS configuration status
- Error messages or logs

## Feature Requests

Use the feature request template and include:
- Clear description of the feature
- Use case and motivation
- Proposed implementation (if you have ideas)

## Questions?

Feel free to open a discussion or check the video tutorials for setup help.
